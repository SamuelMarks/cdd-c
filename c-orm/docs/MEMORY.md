# c-orm Memory Ownership & Management Rules

Because `c-orm` is written in standard C89 and heavily leverages dynamic struct hydration via `cdd-c`, explicit and predictable memory management is required to avoid leaks.

## Rules

### 1. Allocations

*   **Strings (`char *`)**: Are dynamically allocated via `strdup()` during database retrieval.
*   **Nullable Primitives (`int32_t *`, `float *`)**: Are allocated via `malloc()` during retrieval if a `NULL` value is not present in the column.

When you pass a struct *into* `c_orm_insert` or `c_orm_update`, `c-orm` only reads the pointers. It does **not** take ownership of your memory. You must manage any dynamically allocated strings or values you supply for insertion yourself.

### 2. Retrieval & Hydration

When retrieving rows (via `c_orm_find_by_id`, `c_orm_find_all`, etc.), `c-orm` dynamically allocates inner pointers for strings and nullable types.

*   You must provide an existing, uninitialized or zero-initialized memory structure (like `struct Users` or `struct Users_Array`).
*   When you are done with a hydrated struct, you **must** call the corresponding free helper function generated by `cdd-c`.

### 3. Cleanup Helpers

Every struct mapped via `cdd-c` comes with two cleanup helper functions defined in `Models.h`.

#### Single Row Freeing: `<Type>_free`

For a single struct:
```c
struct Users u;
memset(&u, 0, sizeof(u));
c_orm_find_by_id_int32(db, &Users_meta, 1, &u);

/* Use u.username, *u.score, etc. */

Users_free(&u); /* Frees internal pointers, zeroing them out. Does NOT free the `u` stack value itself. */
```

#### Array Collection Freeing: `<Type>_Array_free`

For array results:
```c
struct Users_Array arr;
memset(&arr, 0, sizeof(arr));
c_orm_find_all(db, &Users_meta, &arr);

/* Iterate and use `arr.data[i]` */

Users_Array_free(&arr); /* Frees internal pointers for each element, and then `arr.data` buffer */
```

These functions are strictly idempotent. Calling them on a zeroed or already-freed struct is safe. However, always run your final program via Valgrind or AddressSanitizer (ASAN) to verify no edge cases exist.
